## 3. Intellectual Property

### 3.1 Client Deliverables

Upon receipt of full payment, Client owns all intellectual property created specifically for this project, including:
- Custom code written for Client
- Custom designs created for Client
- Custom content created for Client
- Project-specific documentation

Client may use, modify, and distribute these deliverables as they see fit.

### 3.2 Service Provider Retains

Service Provider retains ownership of:
- Pre-existing frameworks, tools, libraries, and methodologies
- Templates and reusable code patterns
- General business processes and knowledge
- Any open-source components with appropriate licensing

Client may not reuse, resell, or redistribute Service Provider's tools, frameworks, or processes.

### 3.3 Third-Party Components

This project may include third-party libraries and open-source code. All such code is provided under its original license:
- WordPress and WordPress plugins (GPL)
- {Other tools and their licenses}

Client agrees to comply with all applicable open-source licenses.

### 3.4 Modifications

If Client modifies the deliverables after handoff, Client is responsible for those modifications. Service Provider is not liable for issues introduced by Client's changes.

---

**Explanation:**

This structure is standard and protects both parties:

- **Client gets:** Everything custom-built for their project
- **Service Provider keeps:** Tools and frameworks that can be used for other clients
- **Both understand:** What code came from where and who can use it

**Why This Matters:**

- **Client satisfaction:** They own what they paid for
- **Your efficiency:** You can reuse your tools for other projects
- **Clear boundaries:** No disputes about ownership later
- **Industry standard:** This is how most service providers handle IP

---

**Variants:**

### If Client Wants Everything

> Service Provider assigns all intellectual property, including tools and frameworks, to Client. Client owns everything created during this project.

**Impact:** Charge significantly higher (50-100% premium) since you can't reuse your tools.

### If Client Wants Source Code Escrow

> In the event Service Provider becomes unable to maintain this project, Service Provider agrees to provide Client with all source code and documentation. This is held in escrow and released only if Service Provider ceases business operations.

**Impact:** Low additional cost. Good for client peace of mind on long-term projects.

### If This Is a Template

> Service Provider is building this as a {WordPress theme / React template / etc.} that may be reused for other clients. Client's unique {content / data / branding} belongs to Client. The underlying {theme / template / framework} remains Service Provider's intellectual property.

**Impact:** Lower client cost since they're getting a proven template, not custom-built.

---

**Common Objections:**

**"We want exclusive ownership of everything."**

Response: "That's standard for fully custom builds. For this project, I'm using established tools that save us time and cost. You own your custom content and code; I own my tools."

**"What about the code you wrote for us?"**

Response: "It's yours. You can modify it, hire someone else to maintain it, even hire me back to work on it. You own it. I just keep my development tools and methodologies."

**"Can you use our code for other clients?"**

Response: "No. Custom code I write for you stays yours. I can use my frameworks and tools for other clients, but not your specific code or your designs."
