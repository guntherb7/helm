# Scope Change Communication Template

Use when a client requests work outside the original contract scope.

## Structure

Subject: {PROJECT_NAME} - Scope Change Discussion

Hi {CLIENT_CONTACT},

During our work on {specific deliverable}, {describe what came up that's outside scope}. This wasn't part of our original agreement, so I wanted to flag it before moving forward.

**What's being requested:**
{describe the additional work clearly}

**Impact:**
- Additional time: {estimate}
- Additional cost: ${amount}
- Impact on timeline: {none / delays X by Y days}

**Options:**
1. Add this to the current project as a change order (+${amount})
2. Include it in a future phase
3. Skip it for now

I want to make sure we're on the same page before I start on this. Let me know how you'd like to proceed.

Best,
{YOUR_NAME}

## Notes
- Always flag scope changes before doing the work
- Frame it as protecting the client's budget, not refusing work
- Provide options, not ultimatums
- If it's truly small (< 1 hour), use judgment on whether to flag or absorb
